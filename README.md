## Repository Structure

The repository is organized into the following directories:

- **`Deterministic-ID-ARX-AND-RX`**: Code and data for impossible differential distinguishers on ARX and AndRX designs.
- **`Deterministic-ZC-ARX-AND-RX`**: Code and data for zero-correlation distinguishers on AndRX designs.
- **`ID-KeyRecovery-ANDRX`**: Code and data for key recovery attacks using impossible differentials on AndRX designs.

### Files in Each Folder
Each folder for a specific cipher contains two primary files:
- `distinguisher.mzn`: The MiniZinc model for the distinguisher used in the attack.
- `data.dzn`: The data file that provides input parameters for the distinguisher model.

## Requirements

To use the tools provided in this repository, ensure that you have the following software installed:

- **[Python 3](https://www.python.org/downloads/)**: Required for running Python scripts.
- **[MiniZinc](https://www.minizinc.org/)**: Used to compile and solve constraint programming (CP) models.
- **[latexmk](https://www.latex-project.org/)**: Generates visual representations of attacks in LaTeX.
- **[Sagemath](https://www.sagemath.org/)**: Used for generating the constraints using SBox Analyzer for the CP models.
- **[Or-Tools](https://developers.google.com/optimization)**: Used to solve our CP models.

## Usage
This repository presents tools for finding ID distinguishers on ARX and AndRX ciphers, ZC distinguishers on AndRX ciphers, and ID key recovery attacks on AndRX ciphers. Follow the instructions below to utilize our tools effectively.

### Finding Distinguishers (ID or ZC)

To find ID or ZC distinguishers for ARX or AndRX ciphers, use the following command:

```bash
minizinc --solver ortools -p 8 distinguisher.mzn data.dzn
```

- **`distinguisher.mzn`**: This file contains the constraint programming (CP) model to find the distinguisher in `.mzn` format.
- **`data.dzn`**: This file contains the data for the distinguisher parameters, including:
  - `RD`: Length of the distinguisher.
  - `MD`: Merging point (in the case of indirect contradiction).

The tool will identify the distinguisher and print it in the terminal. To save the distinguisher output to a text file, use:

```bash
minizinc --solver ortools -p 8 distinguisher.mzn data.dzn > output.txt
```

Alternatively, you can utilize the Python interface of Or-Tools to solve and process the results, which will help you generate the shape of the attack:

```bash
python3 distinguisher.py
```

### Finding ID Key Recovery Attacks

To find ID key recovery attacks for AndRX ciphers, run the following command:

```bash
minizinc --solver ortools -p 8 attacki.mzn data.dzn > output.txt
```

- **`attacki.mzn`**: This file contains the CP model for finding ID key recovery in `.mzn` format.
- **`data.dzn`**: This file contains the data for the attack parameters, including:
  - `RD`: Length of the distinguisher.
  - `MD`: Merging point (in the case of indirect contradiction).
  - `RB`: Length of the upper part.
  - `RF`: Length of the lower part.

The tool will identify the key recovery attack and save it in `output.txt`. Alternatively, you can use the following command to identify the ID key recovery attack and generate its shape:

```bash
python3 attacki.py -RB x -RD x -MD x -RF x
```

We give a few examples, but the same applies to other applications as well.

## Encoding AND, Modular Addition and Other Building Block Functions
 
The S-box Analyzer [[1]](https://tosc.iacr.org/index.php/ToSC/article/view/9858) is an open-source tool developed to encode the differential,linear, differential-linear, and integral properties of S-boxes using MILP, SMT/SAT, andCP models efficiently. In our paper, the CP constraints for bit-wise deterministic differential/linear propagation through Modular Addition operation can be automatically derived with an extended version of the S-boxAnalyzer tool. The corresponding functions in S-box Analyzer for this task are as follows.“-1” means the difference/linear mask can be either 0 or 1.

### Constructing the CP Constraints Presented in Proposition 4

Using [Sbox Analyzer](https://github.com/hadipourh/sboxanalyzer) tool, we model the CP constraints describing all valid bit-wise deterministic differential transitions through the full adder (Proposition 4) as follows:

```Python

sage: sa = SboxAnalyzer(sb)
```
